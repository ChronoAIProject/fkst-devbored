Personas are prompts, not code — each is a lens paragraph plus a model and effort setting, materialised into the loop's config on save. Editing one here never touches platform package internals; it writes to this deployment's roster. Changes apply on the next supervise start, not to debates already in flight.

Flows are the loop's routing rules, not a workflow engine you drive by hand — you set the shape, the loop executes it autonomously. The merge gate's checks are machine facts and deliberately not persona-configurable: no lens can vote CI green.

Scores answer whether a persona is earning its seat. A lens that never objects adds nothing; one that objects and is always overruled is noise; one whose objections keep getting upheld — or that keeps catching what others miss — is doing the work. Measured over merged outcomes, not self-report.
